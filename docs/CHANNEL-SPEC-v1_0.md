## §1 Channel-fork rule

One source product → **separate listing assets per channel** (CLAUDE-Publishing §5.1). Amazon and Etsy reward different signals; never broadcast one listing unmodified. Each channel gets its own title, description, keywords/tags, and correctly-sized files. The `products` row is per-channel (DATA-SCHEMA §4.2).

---

## §2 Universal asset standards

Applies to every print and printable file unless a channel overrides:

- **Resolution:** 300 DPI (images and rendered PDFs).
- **Fonts:** embedded or outlined in the PDF — never rely on system fonts.
- **Interior:** single print-ready PDF, flattened.
- **Color:** KDP accepts RGB; use a CMYK-safe palette to avoid print shifts. Keep the locked brand palette print-tested.
- **Bleed (full-bleed print):** 0.125" (3.2 mm) on each outside edge.
- **Margins (print interior):** outside ≥ 0.25"; inside/gutter scales with page count (≈0.375" for low page counts, more as pages increase).
- **Generated by:** P08 (interiors via WeasyPrint), P09 (covers). QC at P11/P25 before any of this is packaged.

## §3 Trim sizes & formats by product type

| Product type | Print trim | Digital format | Notes |
|---|---|---|---|
| Planner / journal / logbook | 6×9 or 8.5×11 | 8.5×11 PDF | 6×9 for grab-and-go, 8.5×11 for writing room |
| Coloring book | 8.5×8.5 (square) | — | Square stands out; **single-sided pages** so color doesn't bleed through |
| Activity / puzzle | 8.5×11 or 8.5×8.5 | 8.5×11 PDF | — |
| Workbook | 8.5×11 | 8.5×11 PDF | room for working |
| Printable (digital-only) | — | 8.5×11 (US Letter) + A4 optional | 300 DPI PDF; home-printed, usually no bleed; offer A4 for non-US buyers |

Trim choice is set in the blueprint (P07) and must match across the interior, cover, and listing.

## §4 Etsy (P13) — Open API v3, automated after approval

**Flow (create as draft → finalize → activate):**
1. Create draft listing with core fields (title, description incl. §9 COMPLIANCE disclosure line, price, quantity, type = digital).
2. Set **category = Digital** and the **"Designed by seller"** attribute; tick "I used AI-generative technology."
3. Upload listing **images** (mockups/previews) and the **digital file(s)**.
4. Apply **tags** (up to 13) and materials/attributes.
5. Update listing to **active** (or leave as draft if human wants a final glance — default: active on approval, since approval already happened at §9.2).

**Constraints to respect:**
- Up to **13 tags**; each ≤ 20 chars. Use the channel-forked keyword set, no stuffing (COMPLIANCE §5.6).
- Digital files: respect Etsy's current per-file size/count limits (verify; historically several files, each capped — keep printable PDFs optimized).
- Mockup images must accurately represent the file (COMPLIANCE §4.2).

**Fees (verify current):** ~$0.20 listing fee (renews ~4 months/per sale) + ~6.5% transaction + payment processing.

**Writes to `listings`:** channel='etsy', external_id (listing id), listing_url, price, disclosure_applied, status='live', published_at.

## §5 Payhip / Gumroad (P14) — owned storefronts, automated after approval

**Flow:**
1. Create product (title, description incl. disclosure line, price).
2. Upload the digital file(s) (same printable PDFs as Etsy, or channel-tuned).
3. Upload cover/preview images.
4. **Enable email capture / list opt-in** — owning the customer is a first-class goal (CLAUDE-Publishing §5.3).
5. Publish; capture the public URL.

**Notes:**
- Same IP/content screens and disclosure apply (COMPLIANCE §4–§5).
- **Fees (verify current):** Payhip free plan ~5% / paid tiers down to 0%; Gumroad ~10% flat. Choose tier by volume; log in COST-LEDGER.
- Owned channels have **no velocity caps** — but the no-near-duplicate rule still applies (QUALITY-STANDARDS / COMPLIANCE §3.4).

**Writes to `listings`:** channel='payhip'|'gumroad', external_id, listing_url, price, disclosure_applied, status='live', published_at.

## §6 KDP (P15) — package only, MANUAL upload

**P15 never uploads to KDP** (CLAUDE-Publishing §3.1). It assembles a package; a human publishes. The package contains:

- [ ] Print-ready **interior PDF** (correct trim, 300 DPI, embedded fonts, correct margins/gutter).
- [ ] **Cover PDF** — single wraparound (back + spine + front). Spine width is a function of page count and paper type — compute via KDP's cover calculator; 0.125" bleed all around.
- [ ] **Metadata sheet:** title, subtitle, author/brand, description (channel-forked), 7 keywords, 2 categories, price.
- [ ] **Disclosure note:** which elements are AI-generated → for the KDP AI-Content declaration (COMPLIANCE §2.1).
- [ ] **Flags:** low-content box yes/no (COMPLIANCE §2.3); ISBN needed? (medium-content yes).
- [ ] **Manual checklist** for the human: trim, ≥24 pages, low-content box, AI declaration, pricing/royalty band.

**Royalty note (verify):** since Jun 2025, print royalty under $9.99 is 50% (was 60%). Price with this in mind (§7).

**Writes to `listings`:** only **after the human confirms publish** — channel='kdp', external_id (ASIN), listing_url, price, disclosure_applied, status='live', published_at. Until then status stays 'pending'.

---

## §7 Pricing positioning per channel

- **KDP print:** account for the 50% band under $9.99; differentiated products can sit above the commodity price (QUALITY-STANDARDS price-headroom criterion) where the niche supports it.
- **Etsy / Payhip / Gumroad digital:** near-100% margin. Price for *value*, not cost — the differentiation justifies above-commodity pricing. Don't race generic printables to the bottom.
- Same product, different channel → price may differ (digital vs print, fee structures). Record actual price per `listings` row.

---

## §8 Publish ledger (P16)

Every successful publish writes one `listings` row per channel (fields per §4–§6). On failure: status='failed' + note, surface to human. Retiring a listing (P26) sets status='retired'. The ledger is the join point for `tracking` (P17) — metrics attach to listings, not products.

---

## §9 Publish gating (recap)

- Nothing publishes without passing Gate 2 (Safety), Gate 3 (Quality), and human Approve (CLAUDE-Publishing §4.4, §9.2).
- Etsy + Payhip/Gumroad: auto-publish on approval (P13/P14).
- KDP: package on approval; **human uploads** (P15). Ledger row only after human confirms live.
